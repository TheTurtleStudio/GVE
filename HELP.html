Introduction to GVE

1. What is Gevvy?

Gevvy (Gusha Virtual Environment or GVE, just sounds like Gevvy to me when I said it aloud) is just like Python's IDLE, except you can have predefined libraries that will be imported on execution of the program and also predefined "functions" that are just pieces of code that execute upon starting the program. You make them yourself and it sounds pretty useless but it's pretty neat; you can use them to predefine classes which can help you better manage your workflow.

3. Can my Gevvy function access another Gevvy function?

Yes! Gevvy functions can access eachother, allowing for collaboration or additions to Gevvy functions from people who choose to use this. If you stick to the layout displayed in my example function in the functions folder that'd be really cool of you. Not required but it'd be nice if you end up sharing your functions.

4. What are all these files for?

"libraries.txt" contains all the predefined libraries. Add or remove whatever libraries you want in there, remember to separate them by putting them on their own line. The folder named "functions" contains a text file with each of our Gevvy functions inside of it. By now you've probably read all about them. If you need more instruction and guidance on these, there's a function called "exampleFunction.txt" that may help clear some confusion.

5. How do I access my functions? How do I use them?

Whatever you put in the file containing your Gevvy function will just be executed. If you define a class it'll be defined, if you just put a print statement it'll just print. It's nothing revolutionary, if you made a function just call it, anything you input into the terminal is executed in the same scope as all the Gevvy functions.
